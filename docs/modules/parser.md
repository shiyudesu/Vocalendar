# Module: Parser

## 1. 模块定位

自然语言 → 结构化事件草稿。纯函数/服务层，无 HTTP、无 DB、无 UI。

本模块职责：

- 时间表达式解析与归一化
- 实体抽取（location、participants、title）
- 缺失字段计算与补问提示生成
- span 消解与优先级处理

不属于本模块：

- HTTP 路由
- 数据库存取
- 语音转写（Voice 模块）

## 2. 产品目标与范围

### 已交付（文本创建）

- 跑通"文本输入 → 事件草稿 → 用户确认 → 创建成功"的最小闭环
- 识别高频自然语言文本表达模式和主要失败原因
- 范围内：文本输入、事件主体/日期/开始时间基础解析、结构化确认

### 待交付

#### 范围查询解析

- 目标：补齐最基础的自然语言查询能力
- 范围内：今日 / 明日 / 本周 / 关键词 范围解析
- 验证点：用户是否会把查询作为创建后的高频辅助动作

#### 修改意图识别

- 目标：让用户可以修改已创建但信息不准确的事件
- 范围内："把...改成..."修改意图解析、时间/标题/地点字段变更提取
- 验证点："改时间"是否是最常见、最有价值的修改操作

#### 删除意图识别

- 目标：让用户可以删除错误或过期的事件
- 范围内：删除意图识别、待删事件定位
- 验证点：删除流程是否安全可理解

#### 语音文本适配

- 目标：支持语音转写后的口语化文本解析
- 范围内：口语化表达归一化、语气词过滤
- 依赖：Voice 模块提供转写文本

#### 提醒与重复解析（预留）

- 目标：支持提醒时间和周期性规则解析
- 范围内：提醒偏移词（提前15分钟）、重复规则（每天/每周/每月）

## 3. 接口契约

- **暴露**：函数 `parseDraft(input: ParseDraftInput): DraftResult`
- **依赖**：Schemas（schema 类型）
