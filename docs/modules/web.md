# Module: Web

## 1. 模块定位

Vite React 浏览器入口，用户交互层。

本模块职责：

- 文本/语音输入交互
- 草稿展示、缺失字段补充、确认创建
- 事件列表展示
- API Client 封装

不属于本模块：

- 自然语言解析（Parser 模块）
- HTTP 服务与数据库（API 模块）
- 语音底层采集（Voice 模块）

## 2. 产品目标与范围

### 已交付

- 文本输入页、草稿展示（解析结果/缺失字段/警告/补问提示）

### 待交付

#### 补充字段与确认创建

- 目标：允许用户补充缺失字段并确认创建事件
- 范围内：补充输入框、字段编辑表单、调用 PATCH /drafts 和 POST /events
- 当前状态：**未实现**。API client 缺少 `updateDraft`、`createEvent` 函数；UI 缺少补充表单和确认按钮

#### 最近列表

- 目标：展示最近创建的事件
- 范围内：调用 GET /events、列表展示
- 当前状态：**未实现**。API client 缺少 `listEvents` 函数；UI 无列表组件

#### 查询页面

- 目标：补齐最基础的事件查询界面
- 范围内：今日/明日/本周快捷查询、关键词搜索、结果列表与摘要
- 验证点：用户是否会把查询作为创建后的高频辅助动作

#### 事件编辑

- 目标：支持修改已创建事件
- 范围内：编辑表单、时间选择器、字段校验、修改确认
- 验证点："改时间"是否是最常见操作

#### 事件删除

- 目标：支持安全删除事件
- 范围内：删除确认弹窗、删除后列表刷新、撤销预留
- 验证点：删除流程是否安全可理解

#### 语音录音组件

- 目标：支持语音输入界面
- 范围内：录音按钮、转写结果展示、纠错编辑、提交
- 依赖：Voice 模块提供录音和转写能力

#### 提醒与重复设置（预留）

- 目标：支持提醒和周期性规则配置
- 范围内：提醒时间设置、重复规则选择器、规则展示

#### 日历同步配置（预留）

- 目标：与现有日历工具连接
- 范围内：账号连接、同步开关、状态反馈

#### 账号体系（预留）

- 目标：完整 Web 产品体验
- 范围内：登录/注册页、个人中心、数据管理
- 阶段：v1.0

#### 移动端适配

- 目标：Web 在移动端可用
- 范围内：响应式布局、触摸优化

## 3. 接口契约

- **暴露**：无（终端消费者）
- **依赖**：Schemas（类型）、API（HTTP 接口）
