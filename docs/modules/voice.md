# Module: Voice

## 1. 模块定位

语音输入链路：录音 → 转写 → 文本 → 复用既有解析流程。

本模块职责：
- 浏览器录音 API 封装
- 语音转写（ASR）接入
- 转写结果展示与纠错交互
- 录音权限管理与失败重试

不属于本模块：
- 自然语言解析（Parser 模块处理转写后的文本）
- HTTP 服务（API 模块）
- 前端页面渲染（Web 模块提供 UI 壳体）

## 2. 产品目标与范围

### 目标
- 在现有文本 CRUD 基础上接入语音输入和转写能力
- 验证语音是否是比文本更高频、更高效的输入入口

### 范围内
- 浏览器录音或受控语音输入
- 转写结果展示与纠错
- 语音转写文本复用既有 drafts/events 流程
- 录音权限、失败重试和必要埋点

### 范围外
- 独立语音指令系统（复用文本解析流程）
- 离线语音识别（首期依赖云端 ASR）

## 3. 接口契约

- **暴露**：函数/组件 `startRecording(): Promise<AudioBlob>`、`transcribe(audio): Promise<string>`
- **依赖**：Schemas（source=voice 字段）、Parser（转写后文本送入解析）、Web（UI 交互）

## 5. 验收标准

- [ ] 浏览器可录音并获取音频
- [ ] 转写结果可展示和编辑
- [ ] 语音输入端到端成功率 > 80%
- [ ] 失败时有明确重试引导

## 6. 风险与阻塞

- 风险：ASR 服务选择（自部署 vs 第三方）→ 对冲：先接入可切换的抽象层
- 阻塞：依赖 Web 模块的输入交互框架；依赖 Parser 对口语化文本的适配
