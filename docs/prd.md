# Vocalendar — 语音日历工具产品需求文档

> 版本：v1.0  
> 日期：2026-05-30  
> 状态：初稿

---

## 1. 产品概述

**Vocalendar** 是一款以语音交互为核心的智能日历管理工具，旨在帮助用户通过自然语言快速完成日历事件的创建、查询、修改与删除，从而显著降低传统手动录入的时间成本，提升日程管理效率。

产品口号：**"说出来，就安排好。"**

---

## 2. 背景与痛点

### 2.1 用户痛点

| 痛点 | 描述 |
|------|------|
| 录入繁琐 | 在移动设备上手动输入事件标题、时间、地点、重复规则等信息步骤多、效率低 |
| 场景受限 | 开车、做饭、运动时双手被占用，无法操作屏幕进行日程管理 |
| 查询不便 | 想快速知道"下周有什么安排"需要多次点击和滚动，体验割裂 |
| 遗漏提醒 | 事件创建后忘记设置提醒，或提醒方式单一导致错过重要日程 |
| 多端不同步 | 手机、电脑、平板上的日程信息不一致，需要反复手动同步 |

### 2.2 市场机会

- 现有日历工具（Google Calendar、Apple Calendar、Outlook）功能强大，但语音交互多为辅助入口，未形成闭环体验
- 语音助手（Siri、Google Assistant）可创建提醒，但日历管理能力浅，无法满足复杂场景
- 专业人群（销售、项目经理、自由职业者）日程密集，对高效录入有强需求

---

## 3. 产品目标

### 3.1 核心目标

1. **语音录入效率提升 80%** — 通过语音创建事件的平均耗时低于 5 秒
2. **语音识别准确率 ≥ 95%** — 在常见日历语义场景下，意图识别与槽位填充准确率达到 95% 以上
3. **多端无缝同步** — 支持 Web、iOS、Android 三端实时同步，延迟 < 1 秒
4. **自然语言全覆盖** — 支持中文自然语言中的时间表达、重复规则、地点、参与者等复杂语义

### 3.2 成功指标

| 指标 | 目标值 |
|------|--------|
| 语音创建事件占比 | ≥ 60% |
| 用户次日留存率 | ≥ 40% |
| 事件提醒触达率 | ≥ 98% |
| 平均事件创建耗时 | ≤ 5 秒 |

## 4. 目标用户与用户场景

### 4.1 核心用户画像

- **忙碌职场人**：日程密集，经常在通勤、会议间隙需要快速记录待办
- **自由职业者**：工作时间灵活，需要频繁调整日程安排
- **学生群体**：课程、考试、社团活动多，偏好轻量化工具

### 4.2 典型使用场景

| 场景 | 用户行为 | 期望结果 |
|------|----------|----------|
| 通勤路上 | "帮我记录明天上午 10 点和客户开会，地点在国贸三期" | 自动创建事件，包含时间、标题、地点 |
| 晨间规划 | "今天我有什么安排？" | 语音播报今日所有事件及时间 |
| 睡前复盘 | "把明天下午的会议改到后天早上" | 准确找到并修改对应事件 |
| 多人协作 | "创建本周五团队聚餐，邀请小明和小红" | 创建事件并发送邀请通知 |
| 周期性事务 | "每周一、三、五晚上 8 点提醒我去健身" | 创建重复事件并设置提醒 |

## 5. 功能需求

### 5.1 语音交互核心模块

#### 5.1.1 语音唤醒与监听

- 支持点击麦克风按钮或长按空格键进入语音输入模式
- 移动端支持摇一摇快速唤醒语音输入（可选）
- 录音过程中实时显示音量波形动画，给予用户反馈
- 支持手动结束录音与自动静音检测结束（VAD）

#### 5.1.2 语音识别（ASR）

- 集成主流语音识别服务（如 Whisper / 讯飞 / 阿里云），支持中文普通话识别
- 支持离线识别基础指令（移动端优先）
- 识别结果实时上屏，允许用户编辑修正

#### 5.1.3 自然语言理解（NLU）

系统需准确解析以下语义要素：

| 语义槽位 | 示例 | 必填 |
|----------|------|------|
| 意图 | 创建、查询、修改、删除、提醒 | 是 |
| 事件标题 | "和客户开会"、"团队聚餐" | 是 |
| 时间 | "明天上午 10 点"、"下周三下午"、"2026 年 6 月 1 日" | 是 |
| 结束时间 | "持续两小时"、"到下午 5 点" | 否 |
| 地点 | "国贸三期"、"线上腾讯会议" | 否 |
| 重复规则 | "每天"、"每周一"、"每月第一个周五" | 否 |
| 提醒时间 | "提前 15 分钟"、"开始前 1 小时" | 否 |
| 参与者 | "邀请小明"、"发给 team@company.com" | 否 |
| 优先级 | "紧急"、"重要"、"低优先级" | 否 |

#### 5.1.4 语音反馈（TTS）

- 操作完成后通过语音播报确认结果，如："已为您创建明天上午 10 点的客户会议"
- 查询类操作直接语音播报内容，如："您今天有 3 个安排：上午 9 点部门早会……"
- 支持在设置中开启/关闭语音反馈，调节语速

### 5.2 日历管理功能

#### 5.2.1 事件 CRUD

- **创建**：支持语音/手动创建事件，包含标题、时间、地点、描述、提醒、重复、参与者、优先级、标签
- **查询**：支持按日/周/月视图查看；支持语音查询特定时间段或关键词
- **修改**：支持时间拖拽调整、语音指令修改、详情页编辑
- **删除**：支持单条删除与批量删除；删除前语音确认

#### 5.2.2 日历视图

| 视图 | 功能描述 |
|------|----------|
| 日视图 | 以时间轴展示当天事件，支持快速语音添加 |
| 周视图 | 横向 7 天时间格，直观展示一周安排 |
| 月视图 | 传统日历网格，显示事件摘要，点击展开详情 |
| 列表视图 | 按时间倒序展示即将发生的事件，适合快速浏览 |

#### 5.2.3 提醒与通知

- 支持多种提醒方式：应用内推送、邮件、短信（可选）
- 提醒时间可自定义：提前 5/10/15/30 分钟、1 小时、1 天等
- 支持重复事件的独立提醒设置
- 错过提醒后支持"稍后提醒"（Snooze）功能

### 5.3 智能辅助功能

#### 5.3.1 语义纠错与确认

- 当 NLU 解析置信度较低时，主动语音确认："您是说要创建明天上午 10 点还是下午 10 点？"
- 支持模糊匹配："把我刚才加的会议删掉" → 系统自动关联最近创建的事件

#### 5.3.2 智能建议

- 根据历史习惯推荐事件时长（如"客户会议"通常 1 小时）
- 时间冲突检测：创建事件时若与其他事件重叠，语音提醒并建议调整
- 根据地点自动估算交通时间，提前提醒出发

#### 5.3.3 快捷指令

支持预设快捷语音指令：

| 指令 | 动作 |
|------|------|
| "我到家了" | 标记当前进行中的外出事件为结束 |
| "会议延期 15 分钟" | 将当前/下一个会议的开始和结束时间都延后 15 分钟 |
| "会议取消" | 删除当前/下一个即将开始的事件 |

### 5.4 多端同步与数据

- 用户账号体系：支持邮箱注册、验证码登录、OAuth（Google/Apple/微信）
- 实时同步：基于 WebSocket 或 SSE 实现多端事件实时同步
- 离线支持：移动端支持离线查看和创建事件，联网后自动同步
- 数据导出：支持导出为 ICS / CSV 格式，便于迁移

## 6. 非功能需求

### 6.1 性能要求

- 语音输入到解析完成响应时间 ≤ 2 秒（正常网络）
- 日历月视图切换渲染时间 ≤ 300ms
- API 接口平均响应时间 ≤ 200ms
- 应用冷启动时间：移动端 ≤ 2 秒，Web 端首屏 ≤ 1.5 秒

### 6.2 可用性要求

- 核心功能（语音创建、查看日程）支持纯语音完成，无需看屏幕
- 界面符合 WCAG 2.1 AA 标准，支持字体大小调节
- 支持深色模式，适配系统级主题切换

### 6.3 安全与隐私

- 语音数据仅在必要时上传至识别服务，不支持语音数据的持久化存储
- 日历事件数据加密存储（AES-256）
- 支持两步验证（2FA）
- 符合 GDPR / 个人信息保护法要求，支持数据删除请求

### 6.4 兼容性

| 平台 | 要求 |
|------|------|
| Web | Chrome 90+、Safari 14+、Edge 90+、Firefox 88+ |
| iOS | iOS 15+、iPhone / iPad 兼容 |
| Android | Android 10+（API 29+） |

## 7. 技术架构

本项目采用 Monorepo 架构，使用 pnpm workspace + Turbo 管理：

```
vocalendar-monorepo/
├── packages/
│   ├── schemas/        # 共享数据模型与类型定义（Zod / TypeScript）
│   ├── api/            # 后端 API 服务（RESTful + WebSocket）
│   ├── web/            # Web 前端（Vite + React/Vue）
│   └── mobile/         # 移动端（Capacitor 跨端框架，iOS & Android）
├── docs/               # 产品文档与技术文档
└── turbo.json          # 构建流水线配置
```

### 7.1 关键技术选型

| 层级 | 技术方案 | 说明 |
|------|----------|------|
| 语音识别 | Whisper API / 讯飞语音听写 | 云端 + 离线混合方案 |
| 自然语言处理 | 自研规则引擎 + LLM 辅助 | 核心槽位用规则保证准确率，复杂语义用 LLM |
| 前端框架 | React 18 + TypeScript | Web 与移动端共用组件库 |
| 移动端 | Capacitor | 一套代码输出 iOS/Android，支持原生插件调用 |
| 后端框架 | Node.js / Hono / Express | 轻量高效，支持 WebSocket 实时推送 |
| 数据库 | PostgreSQL + Redis | 主数据库存储，Redis 缓存与消息队列 |
| 实时同步 | WebSocket / SSE | 事件变更实时推送至多端 |
| 部署 | Docker + CI/CD | 支持自动化测试与部署 |

## 8. 数据模型（核心）

### 8.1 Event（事件）

```typescript
interface Event {
  id: string;                    // UUID
  userId: string;                // 所属用户
  title: string;                 // 事件标题
  description?: string;          // 详细描述
  startTime: Date;               // 开始时间
  endTime?: Date;                // 结束时间
  allDay?: boolean;              // 是否全天事件
  timezone: string;              // 时区（默认用户本地时区）
  location?: string;             // 地点
  recurrence?: RecurrenceRule;   // 重复规则
  reminders: Reminder[];         // 提醒设置
  attendees?: Attendee[];        // 参与者
  priority: 'low' | 'normal' | 'high';
  tags?: string[];               // 标签
  source: 'voice' | 'manual';    // 创建来源
  createdAt: Date;
  updatedAt: Date;
}
```

### 8.2 RecurrenceRule（重复规则）

```typescript
interface RecurrenceRule {
  frequency: 'daily' | 'weekly' | 'monthly' | 'yearly';
  interval?: number;             // 间隔（默认 1）
  byWeekDay?: number[];          // [1,3,5] 表示周一三五
  byMonthDay?: number[];         // 每月第几天
  until?: Date;                  // 结束日期
  count?: number;                // 重复次数
}
```

### 8.3 Reminder（提醒）

```typescript
interface Reminder {
  id: string;
  eventId: string;
  minutesBefore: number;         // 提前分钟数
  method: 'push' | 'email' | 'sms';
  sentAt?: Date;                 // 实际发送时间
}
```

## 9. 交互流程示例

### 9.1 语音创建事件

```
[用户] 长按麦克风 → 说话："明天下午 3 点去 dentist，提前半小时提醒我"
      ↓
[系统] 录音结束 → ASR 识别文本 → NLU 解析
      ↓
[系统] 语音确认："已理解：明天 6 月 1 日下午 3 点的 dentist，提前 30 分钟提醒，对吗？"
      ↓
[用户] "对的" / 点击确认
      ↓
[系统] 创建事件 → 同步至所有设备 → 语音播报："已添加"
```

### 9.2 语音查询日程

```
[用户] "下周我要出差吗？"
      ↓
[系统] 解析时间范围（下周）+ 关键词（出差）
      ↓
[系统] 语音播报："下周您有 2 个和出差相关的事件：
       周一上午飞上海，周四下午返程。"
```

## 11. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 语音识别在嘈杂环境准确率低 | 高 | 支持手动修正；优化 VAD 算法；提示用户在安静环境使用 |
| 自然语言时间表达歧义 | 高 | 建立时间表达规则库；低置信度时主动确认 |
| 多平台实时同步延迟 | 中 | 使用 WebSocket + 本地乐观更新；设计冲突解决策略 |
| 第三方语音识别服务成本 | 中 | 评估自研轻量模型；用户分层，高频用户使用离线识别 |
